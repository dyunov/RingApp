Contribute

To contribute, you will either need to register here on this site, or else have a GitHub account.

For bug reports, feature requests, or problems with the website itself, use the RingApp GitHub issue tracker.

For suggestions or corrections to the database content, use the dart_data GitHub issue tracker.

If for some reason GitHub is not for you, you can always use the form below to make any suggestion.

After getting feedback at your issue, you may wish to start developing your pull request. Here is a brief guide for how to do that for a GitHub project. Doing this just makes it quicker for your change to get reviewed and adopted, since an admin will not have to do it for you.

Reminders



To help maintain the quality of content and integrity of the database, please keep these points in mind:
  • We might not accept every suggestion.
  • Did you check the database to see if your ring example or property is already there? (Accidents will happen and we can deal with it, but it's best to avoid that altogether.)
  • It is really important to have good citations! (Incorrect proofs and claims exist, unfortunately.)
  • Multiple high-quality citations are helpful. (Readers will have more places to look.)
